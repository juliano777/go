## Go Modules

Criar o diretório `app-test` e em seguida acessá-lo:
```bash
mkdir app-test && cd app-test
```  
  
Inicializar o módulo Go:
```bash
go mod init app-test
```
Passo necessário para marcar o diretório atual como a raiz de um módulo Go e
inicializa o sistema de gestão de dependências do projeto.  
  
Listar o conteúdo do diretório:
```bash
ls
```
```
go.mod
```
Foi criado o arquivo `go.mod`. Esse arquivo é o registro das dependências
necessárias.  

Exibir o conteúdo do arquivo `go.mod`:
```bash
cat go.mod 
```
```
module app-test

go 1.25.4
```
`module app-test`: caminho do módulo; define o nome do módulo. É usado para
todas as importações de pacotes do seu próprio código, que nesse caso é
simplesmente `app-test`.  
`go 1.25.4`: vesrsão de Go; especifica a versão mínima da _toolchain_ Go que
este módulo requer para compilar e funcionar corretamente.   
  
Checar a versão de Go que está instalada:
```bash
go version
```
```
go version go1.25.4 darwin/arm64
```

Criar um simples arquivo Go:
```bash
vim app-test.go
```
```go
package main

import "fmt"

func main(){
    fmt.Println("Long live Rock in Roll!")
}
```  
  
Rodar o arquivo:
```bash
go run app-test.go
```
```
Long live Rock in Roll!
```  
   
O comando anterior não gerou um binário:
```bash
ls
```
```
app-test.go go.mod
```
Go **não** é uma linguagem interpretada, mas sim compilada.  
Apesar de não ter gerado um binário houve sim um processo de compilação para
poder rodar o código.  
  
Compilar o arquivo de código:
```bash
go build
```  
  
Novamente listar arquivos:
```bash
ls
```
```
app-test    app-test.go go.mod
```
Agora nota-se um novo arquivo, `app-test`, o qual foi gerado pelo processo de
compilação. Ou seja, é um binário executável.  
  
Executar o binário gerado:
```bash
./app-test
```
```
Long live Rock in Roll!
```  
  
Similarmente ao compilador [gcc](https://gcc.gnu.org/) é possível especificar
o nome (_output_) do arquivo binário a ser gerado:
```bash
go build -o outronome
```  
  
Listar arquivos no diretório:
```bash
ls
```
```
app-test    app-test.go go.mod      outronome
```  
Conforme especificado na compilação, `outronome` foi o novo binário gerado.  

Executar o binário:
```bash
./outronome
```
```
Long live Rock in Roll!
```
   
Compilando dizendo de forma explícita qual arquivo Go deve ser usado:   
```bash
go build app-test.go
```  
  
Listando os arquivos do diretório com o comando `tree`:
```bash
tree .
```
```
.
├── app-test
├── app-test.go
├── go.mod
└── outronome
```
Aqui nota-se que todos estão no mesmo nível.
